Amazon Toolkit

The Amazon Toolkit extension for Amazon Cloud9 that enables you to interact with Amazon Web Services (Amazon). See the user guide for complete documentation.

Try the Amazon Code Sample Catalog to start coding with the Amazon SDK.

Amazon Explorer

The Amazon Explorer provides access to the Amazon services that you can work with when using the Toolkit. To see the Amazon Explorer, choose the Amazon icon in the Activity bar.

Developer Tools

The Developer Tools panel is a section for developer-focused tooling curated for working in an IDE. The Developer Tools panel can be found underneath the Amazon Explorer when the Amazon icon is selected in the Activity bar.

CDK Explorer

The Amazon CDK Explorer enables you to work with Amazon Cloud Development Kit (CDK) applications. It shows a top-level view of your CDK applications that have been sythesized in your workspace.

With the CDK explorer, you can navigate the CDK application's infrastructure stacks, resources, and policies.

For full details see the Amazon CDK Explorer in the user guide.

Amazon CodeWhisperer

Amazon CodeWhisperer provides inline code suggestions using machine learning and natural language processing on the contents of your current file. Supported languages include: Java, Python and Javascript.

Once enabled, CodeWhisperer will provide code suggestions automatically and can also be requested manually using option+c (mac) / alt+c (PC). To accept a suggestion and add it to your file, press Tab, Enter or click on it. To dismiss a suggestion, press escape or keep typing.

For more information, see Amazon CodeWhisperer in our user guide.

Amazon Serverless Applications

The Amazon Toolkit enables you to develop Amazon serverless applications locally. It also provides Inline Actions in Cloud9 to do the following:

  • Use SAM (serverless application model) templates to build and debug your locally developed Amazon serverless applications.
  • Run selected Amazon Lambda functions.

To start debugging with a SAM template, click the Add Debug Configuration Inline Action in the template file.

The Inline Action indicator in the SAM template allows you to add a debug configuration for the serverless application.

Alternatively, you can run and debug just the Amazon Lambda function and exclude other resources defined by the SAM template. Again, use an Inline Action indicator for an Amazon Lambda-function handler. (A handler is a function that Lambda calls to start execution of a Lambda function.)

The Inline Action indicator in the application file lets you add a debug configuration for a selected Amazon Lambda function.

When you run a debug session, the status and results are shown in the Amazon Toolkit output channel. If the toolkit does not have an open Amazon Toolkit output channel, one can be created with the New Tab button.

After a local run is complete, the output appears in the OUTPUT tab.

When you're satisfied with performance, you can deploy your serverless application. The SAM template is converted to a CloudFormation template, which is then used to deploy all the application's assets to the Amazon Cloud.

Supported runtimes

The Toolkit local SAM debugging feature supports these runtimes:

  • JavaScript (Node.js 12.x, 14.x)
  • Python (3.7, 3.8, 3.9, 3.10)

For more information see Working with Amazon Serverless Applications in the user guide.

Amazon: Commands

The Toolkit provides commands (prefixed with Amazon:) to the Amazon Cloud9 Go to Anything panel, available by clicking the search bar and typing "." or via hotkey.

OS Hotkey
Windows CTRL-.
macOS CMD-.

Amazon Command Description
Amazon: About Toolkit Displays information about the Amazon Toolkit.
Amazon: Add SAM Debug Configuration Creates an aws-sam Debug Configuration from a function in the given source file
Amazon: Connect to Amazon Select or create a connection to Amazon.
Amazon: Create a new Issue on Github Opens the Amazon Toolkit's New Issue page on Github.
Amazon: Create Credentials Profile Creates an Amazon credentials profile.
Amazon: Create Lambda SAM Application Generates code files for a new Amazon serverless Lambda application. For more information, see Creating a Serverless Application in the user guide.
Amazon: Create new CloudFormation Template Creates a new starter Cloudformation Template
Amazon: Create new SAM Template Creates a new starter SAM Template
Amazon: Deploy SAM Application Deploys a local serverless application to an Amazon account. For more information, see Deploying a Serverless Application in the user guide.
Amazon: Edit Credentials Opens the ~/.aws/credentials or ~/.aws/config file for editing.
Amazon: Edit SAM Debug Configuration Shows a tool that helps you create, edit, run, and debug a SAM launch config (type:aws-sam).
Amazon: Detect SAM CLI Checks whether the Toolkit can communicate correctly with the Amazon SAM CLI that is installed.
Amazon: Show or Hide Regions Adds or removes Amazon Regions in the Amazon Explorer.
Amazon: Sign out Disconnect the Toolkit from the current Amazon connection.
Amazon: Submit Quick Feedback... Submit a private, one-way message and sentiment to the Amazon Toolkit dev team. For larger issues that warrant conversations or bugfixes, please submit an issue in Github with the Amazon: Create a New Issue on Github command.
Amazon: Toggle SAM hints in source files Toggles Amazon SAM-related Inline Actions in source files
Amazon: View Toolkit Logs Displays log files that contain general Toolkit diagnostic information.
Amazon: View Quick Start Open this quick-start guide.
Amazon: View Toolkit Documentation Opens the user guide for the Toolkit.
Amazon: View Source on GitHub Opens the GitHub repository for the Toolkit.
